원문은 대규모 Kubernetes 인프라 변경을 Claude Code 같은 AI와 "동료로" 수행하기 위한 구조적 방법론을 제시한다. 본 프로젝트는 이 방법론을 **OpenShift + OpenShift AI + GitOps** 도메인에 적응한다.

## 🔒 불변 원칙 (IMMUTABLE)

이 원칙들은 수정 불가. 변경이 필요하다고 판단되면 AI는 사람에게 제안만 할 것.

### 1. AI는 판단, 실행은 선언형 도구
- AI가 `oc apply`를 프로덕션 리소스에 직접 치지 않는다.
- AI는 매니페스트를 작성하고, **ArgoCD가 적용**한다.
- 예외: Day-0 부트스트랩, 읽기 전용 진단 명령.

### 2. 판단의 여지를 줄인다
- 버전·네임스페이스·도메인·채널은 `infra/` 파일로 고정.
- AI가 "추정"해야 하는 숫자는 원칙적으로 없다.
- 모호하면 실행 대신 Open Question을 기록한다.

### 3. 상태는 파일에 산다
- 대화 컨텍스트는 휘발성. 의미 있는 상태는 전부 `claude-context/`에 기록.
- 다음 세션이 **cold-start에서 완전히 재구성** 가능해야 한다.
- 검증 방법: 세션을 강제 종료 후 다른 터미널에서 재개. 막힘없이 이어지면 성공.

### 4. 번호 순서는 강제력이다
- `runbooks/`는 `00 → 10 → 20 → ...` 순서를 반드시 따른다.
- 건너뛰기는 명시적으로 `work-plans/`에 기록해야 한다.

### 5. 실패는 데이터다
- 실패 원인은 `work-plans/` 또는 `claude-context/constraints.md`에 누적.
- 재발 방지의 책임은 다음 세션이 아니라 **실패를 겪은 세션**에 있다.

### 6. 계약을 조용히 어기지 않는다
- `guidelines/01-layer-contracts.md`의 계약을 위반하려는 시점에 **중단**.
- 사람에게 (a) 대안 (b) 계약 변경 (c) 일회 예외 중 선택 요청.

## 4계층의 역할과 흐름

```
Layer 1 (work-plans/)       사람이 "왜/어떻게/트레이드오프"를 결정하는 곳
       │  증류
       ▼
Layer 2 (claude-context/)   AI가 매 iteration 읽는 최소 맥락
       │  지시
       ▼
Layer 3 (runbooks/)         실행 단계의 검증 가능한 절차
       │  참조
       ▼
Layer 4 (infra/)            불변의 선언형 정의 (IaC)
```

상위 레이어가 하위 레이어를 **지휘**한다. 역방향은 금지:
- runbook이 work-plan을 덮어쓰지 않는다.
- infra YAML이 runbook의 실행 순서를 바꾸지 않는다.

## 왜 이 구조인가

LLM의 실패 모드를 구조로 막는다:

| LLM 실패 모드 | 방지 장치 |
|---|---|
| 문맥 과적재로 인한 환각 | Layer 2의 증류·최소화 (파일당 500줄 상한) |
| 순서 혼동으로 인한 실행 오류 | Layer 3의 번호 강제 |
| 값 추정으로 인한 일관성 파괴 | Layer 4의 IaC 고정 + 출처 주석 |
| 세션 단절로 인한 상태 상실 | state 파일 + handoff 프로토콜 |
| 계약 위반의 은닉 | 명시적 중단·보고 의무 |

---

## 다른 패턴과의 관계

- **Ralph Loop**: 본 구조는 Ralph의 "상태를 파일에 밀어넣는" 철학과 동일. 이 프로젝트에서 Ralph Loop 사용은 선택적이지만, 구조는 Ralph-ready다.
- **서브에이전트**: 각 레이어의 탐색·검토에 활용 가능. 단, **상태 갱신 권한은 메인 세션에만**.
- **GitAIOps**: Git = Single Source of Truth. AI는 이 원칙의 실행자이지 예외가 아니다.

## 이 문서의 수정

이 파일을 수정하려면:
1. 사람이 직접 편집
2. 커밋 메시지: `[methodology] <변경 사유>`
3. 변경 전후 요약을 `claude-context/handoff-notes.md` 하단에 기록

AI는 이 파일에 **제안만 가능**. 직접 edit 금지.
